    Daggerfall TEXT.RSC Index[edit <https://en.uesp.net/w/index.php?
    title=Daggerfall_Mod:TEXT.RSC_indices&action=edit&section=1>]

*v1.6 10 March 2004* Created By Brendan

This an index of the entries in the TEXT.RSC file found in the ARENA2
directory of "The Elder Scrolls II: Daggerfall". Each index has a four
digit number from 0000 to 9999. Many index numbers are not used. The
used indexes are listed in ascending numerical order as spans {####-
####} or single entries followed by descriptions {#### = 'foo'},
sometimes with additional single entries or spans appended after a comma
{,}. Descriptions in the form ' #### = "foo" ' have the literal contents
of the entry within the double quotes {""}; in this case the entry would
be: foo. Descriptions may also take the form of headers followed on the
next line by the indexes they pertain to. Additional comments are
contained in parenthesis {('foo')}.

The information here is 99% complete and fairly accurate. On
completeness, the entire file has been explored and documented, but
changes between patched versions are not well researched beyond
differences between the original and the last 2.13 patched version. For
accuracy, a number of the entries are not used in the game, some have
identical index numbers, the text of some entries are identical and it
has not been thoroughly tested which index number is the one used, and
certain error messages encountered in the game regarding "bad text" or
missing entries all introduce a few unknowns. Attempts have also been
made to chart the useage of entries for character biographies, but this
has been left incomplete. Also worth noting is that there are a number
of entries discovered in the TEXT.RSC file included with the 'Betony
Demo' of Daggerfall that have not been included here. Neither have the
'Privateer's Hold Demo' entries been included.

LIST OF USED INDEXES

"####-####" is a continuous, incremental span of used index numbers
including the starting and ending numbers. "####" is a lone entry that
is used but is not part of a span, i.e. neither the index number one
less nor one greater was used. " = 'foo'" appended to any index
number(s) are special cases where 'foo' is a description of a lone entry
or a span worth noting ("foo" does not appear literally). " = VOID"
means that the index number is used but contains no information. ",####"
appended to any index number(s) are lone entries that are part of a set
whose content belongs together, all of which will be followed by a
description. These may take the form of an addendum indented below a
span of used indexes they fall within.
